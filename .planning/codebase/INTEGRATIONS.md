# External Integrations

**Analysis Date:** 2026-04-30

## APIs & External Services

**Signal K Server:**

- Signal K plugin host - Primary runtime integration.
  - Integration method: plugin factory exported from `src/index.ts`.
  - Host APIs used: `app.handleMessage`, `app.setPluginStatus`, `app.setProviderStatus`, `app.readPluginOptions`, `app.savePluginOptions`, `app.getSelfPath`, `app.debug`, and `app.error`.
  - Router integration: `plugin.registerWithRouter` delegates to route registration in `src/routes.ts`.
  - Data path: received deltas are injected back to Signal K via `app.handleMessage`.

**Remote Signal K Edge Link Peer:**

- Encrypted UDP peer - Client/server data transfer between Signal K instances.
  - Integration method: UDP sockets from Node `dgram`.
  - Wire protocols: v1 encrypted payloads in `src/pipeline.ts`; v2/v3 packetized reliable transport in `src/pipeline-v2-client.ts`, `src/pipeline-v2-server.ts`, and `src/packet.ts`.
  - Auth/confidentiality: shared `secretKey` with AES-256-GCM; v3 adds HMAC-authenticated control packets.
  - Operational requirement: both ends must match `protocolVersion`, `secretKey`, `useMsgpack`, `usePathDictionary`, and `stretchAsciiKey`.

**Signal K Admin / Web UI Host:**

- Browser-hosted admin surface - Configuration and runtime management UI.
  - Build integration: `webpack.config.js` exposes `./PluginConfigurationPanel` via Module Federation.
  - Runtime API client: `src/webapp/utils/apiFetch.ts` attaches management auth headers when a token is available.
  - Schema source: UI consumes `src/shared/connection-schema.ts` through `buildWebappConnectionSchema`.

**Management REST API Consumers:**

- Operators, scripts, and the bundled CLI call routes under `/plugins/signalk-edge-link`.
  - Route root: `src/routes.ts`.
  - Route modules: `src/routes/config.ts`, `src/routes/connections.ts`, `src/routes/control.ts`, `src/routes/metrics.ts`, and `src/routes/monitoring.ts`.
  - CLI client: `src/bin/edge-link-cli.ts`.
  - Auth headers: `X-Edge-Link-Token`, `Authorization: Bearer <token>`, and legacy `X-Management-Token`.
  - Rate limit: `RATE_LIMIT_MAX_REQUESTS` per `RATE_LIMIT_WINDOW` in `src/constants.ts`.

## Data Storage

**Databases:**

- None. The plugin has no database dependency.

**File Storage:**

- Signal K plugin data directory - Runtime JSON config files are stored per connection.
  - Files: `delta_timer.json`, `subscription.json`, and `sentence_filter.json`.
  - Helpers: `src/config-io.ts` handles load/save; `src/config-watcher.ts` handles hot reload and legacy migration.
  - Access routes: legacy `/config/:filename` and per-connection `/connections/:id/config/:filename`.
- Packet capture export - `src/packet-capture.ts` stores recent packets in memory and exports PCAP through monitoring routes.

**Caching / In-Memory State:**

- Instance registry - `src/index.ts` owns a `Map` of active instances.
- Per-instance mutable state - `src/instance.ts` owns sockets, timers, watchers, metrics, and subscriptions.
- Client sessions - `src/pipeline-v2-server.ts` tracks remote clients by address and port.
- Source registry - `src/source-replication.ts` tracks source identities observed from incoming deltas.

## Authentication & Identity

**Management API:**

- Token source: plugin option `managementApiToken` or `SIGNALK_EDGE_LINK_MANAGEMENT_TOKEN`.
- Fail-closed option: `requireManagementApiToken` or `SIGNALK_EDGE_LINK_REQUIRE_MANAGEMENT_TOKEN`.
- Comparison: `src/routes.ts` hashes candidate tokens and compares digests with `crypto.timingSafeEqual`.
- Redaction: `src/routes/config.ts` redacts `secretKey` and `managementApiToken` in config responses.

**Transport Auth / Crypto Identity:**

- Payload encryption: `src/crypto.ts` normalizes keys and performs AES-256-GCM encrypt/decrypt.
- Key formats: 32-byte ASCII, 64-character hex, or 44-character base64.
- Optional ASCII stretching: `stretchAsciiKey` derives the AES key through PBKDF2.
- Control plane: v3 control packet auth tags are generated and verified by `src/crypto.ts` and `src/packet.ts`.

**Browser Token Sources:**

- Preferred injection: `window.__EDGE_LINK_AUTH__.token`.
- Local fallback: `localStorage` key `signalkEdgeLinkManagementToken`.
- Query parameter support exists but is disabled by default through `includeTokenInQuery: false` in `src/webapp/utils/apiFetch.ts`.

## Monitoring & Observability

**Prometheus / Metrics:**

- Prometheus text exposition is generated by `src/prometheus.ts` and served from `src/routes/metrics.ts`.
- Signal K metric publishing is handled by `src/metrics-publisher.ts`.
- Runtime metrics state is built by `src/metrics.ts`.

**Alerts and Monitoring Views:**

- Trackers live in `src/monitoring.ts`: packet loss, path latency, retransmission, and alert state.
- Packet capture and inspector live in `src/packet-capture.ts`.
- Operator docs: `docs/metrics.md`, `docs/management-tools.md`, and `docs/security.md`.

**Network Reachability:**

- `ping-monitor` is used by v1 client mode for TCP reachability checks and RTT publishing.
- v2/v3 use protocol telemetry, heartbeats, ACK/NAK behavior, and client-reported network quality.

## Environment Configuration

**Development:**

- Required for most work: Node.js, npm dependencies, and local Signal K test context if doing manual plugin testing.
- Optional token env var: `SIGNALK_EDGE_LINK_MANAGEMENT_TOKEN`.
- Local `.env` is intentionally ignored.

**Production:**

- Secrets live in Signal K plugin settings or environment variables.
- UDP ingress should be restricted to trusted peers.
- Operators should place management routes behind Signal K auth, TLS, or a trusted reverse proxy.

## Webhooks & Callbacks

**Incoming:**

- No third-party webhooks were found.
- Incoming traffic is either HTTP management/API requests or UDP peer packets.

**Outgoing:**

- Signal K delta injection: `app.handleMessage` in pipeline and source dispatch flows.
- Signal K notifications: `AlertManager` in `src/monitoring.ts`.
- UDP send calls to configured peers and bonding links.
