<threat_model>

## Threat Model

| ID     | Threat                                                   | Severity | Mitigation                                                                                                     |
| ------ | -------------------------------------------------------- | -------- | -------------------------------------------------------------------------------------------------------------- |
| T-2-07 | Coalescing drops quick successive threshold updates      | high     | Merge pending thresholds per connection and test multiple metrics plus last-write-wins updates.                |
| T-2-08 | Delayed persistence changes API response semantics       | medium   | Update in-memory thresholds before responding and assert response body contains the new threshold immediately. |
| T-2-09 | Pending timers leak or keep the process alive            | medium   | Delete pending entries after flush and call `unref()` on timers when available.                                |
| T-2-10 | Persistence writes drift to root-level `alertThresholds` | high     | Reuse matching connection persistence logic and test saved options only update the selected connection entry.  |
| T-2-11 | Persistence failure logs expose sensitive configuration  | medium   | Keep existing `app.error` behavior and do not log token or secret values.                                      |

Block condition: any high-severity threat without implemented mitigation blocks completion.

</threat_model>
