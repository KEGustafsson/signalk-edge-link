## Threat Model

| ID     | Threat                                                   | Severity | Mitigation                                                                                      |
| ------ | -------------------------------------------------------- | -------- | ----------------------------------------------------------------------------------------------- |
| T-1-03 | Release guard becomes too broad and blocks valid history | medium   | Scope checks to current-doc markers and known stale source filenames, not all historical notes. |
| T-1-04 | Package publishes stale or missing build artifacts       | medium   | Keep `npm run build` before `npm pack`, check package `files`, and document pack verification.  |
| T-1-05 | Release docs or scripts leak local tokens or credentials | low      | Do not read or print env secrets; keep checks limited to metadata and static docs.              |
| T-1-06 | New script adds dependency or cross-shell fragility      | low      | Use dependency-free CommonJS Node code and run through npm on all platforms.                    |

Block condition: no high-severity threats are introduced. Medium threats must have the mitigations above implemented.

</threat_model>
